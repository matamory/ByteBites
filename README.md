ByteBites is the week 3 tinker activity where students will learn how to build the backend logic for a fake campus food ordering app. There are 4 main portions to this project. During the first phase, we create a system design using the given specs. Afterwards, we move onto scaffolding the code, adding filtering sorting and total logic, and writing comprehensive test cases using pytest. Throughout these phases, the focus is learning how to effectively integrate an AI agent into your workflow. You learn the differences between creating your own agent, as well as the differences between the various modes (plan, ask, agent). This tinker encourages students to think through context windows, prompt enrichment, and output verification. The scope of this project is on the smaller side with 4 main classes including DoodItem, Menu, Transaction and Customer. The smaller scope encourages students to thoroughly verify behavior and iterate on AI outputs. The most challenging aspect of this activity is keeping AI on track, without overengineering solutions.
